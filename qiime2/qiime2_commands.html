QIIME2 Commands

Import the data

Create the folder for end-to-end analysis

mkdir microbiome-fruit
cd microbiome-fruit

Activate conda environment

conda activate qiime2

Import single-end reads

qiime tools import \
  --type MultiplexedSingleEndBarcodeInSequence \
  --input-path forward.fastq.gz \
  --output-path multiplexed-seqs.qza

Demultiplex

Demultiplex the reads

Choosing the amount of cores is based on the machine you are running on

qiime cutadapt demux-single \
  --p-cores 16 \
  --i-seqs multiplexed-seqs.qza \
  --m-barcodes-file metadata-fruits-2.tsv \
  --m-barcodes-column barcode \
  --p-error-rate 0   \
  --o-per-sample-sequences demultiplexed-seqs.qza \
  --o-untrimmed-sequences untrimmed-seqs.qza \
  --verbose

Create a directory to store visualizations

mkdir visualizations

Visualize the quality

qiime demux summarize \
  --i-data demultiplexed-seqs.qza \
  --o-visualization visualizations/demultiplexed-seqs.qzv

Denoise using Deblur

Deblur removes sequencing errors and identifies the true biological sequences present in your samples.

Create a dir for Deblur outputs

mkdir deblur

Apply quality filtering based on the scores This step removes low-quality sequencing reads based on their Phred quality scores.

qiime quality-filter q-score \
 --i-demux demultiplexed-seqs.qza \
 --o-filtered-sequences deblur/demux-filtered-deblur.qza \
 --o-filter-stats deblur/demux-filter-stats-deblur.qza \ 
 --verbose

Making ASVs using Deblur

This step is one of the longest steps. It recommended to use a screen to run Deblur in the background.

Create a screen

screen -S deblur_run

Once you are inside the screen, you can run Deblur. This will allow Deblur to run in the background after leaving the screen.

To detach from the screen, press Ctrl + A. Then D.

screen -ls can be used to see the running sessions.

screen -r name to reattach the screen

To kill the screen inside, enter exit inside the screen.

To kill the screen outside of it, screen -X -S name quit.

Select cutoff based on the demux quality plot (forward reads)

qiime deblur denoise-16S \
  --i-demultiplexed-seqs deblur/demux-filtered-deblur.qza \
  --p-trim-length 190 \
  --p-jobs-to-start 16 \
  --o-representative-sequences deblur/rep-seqs-deblur.qza \
  --o-table deblur/table-deblur.qza \
  --p-sample-stats \
  --o-stats deblur/deblur-stats.qza \
  --verbose

Visualize the summary statistics

qiime metadata tabulate \
  --m-input-file deblur/demux-filter-stats-deblur.qza \
  --o-visualization visualizations/demux-filter-stats-deblur.qzv
qiime deblur visualize-stats \
  --i-deblur-stats deblur/deblur-stats.qza \
  --o-visualization visualizations/deblur-stats.qzv

Generating a Feature Table

The output, table-deblur.qzv, provides information on how many sequences are associated with each sample and feature.

qiime feature-table summarize \
  --i-table deblur/table-deblur.qza \
  --o-visualization visualizations/table-deblur.qzv \
  --m-sample-metadata-file metadata-fruit-2.tsv

Now, let’s map feature IDs to sequences. This will allow use to easily BLAST sequences against the NCBI nt database.

The output, rep-seqs-deblur.qzv contains the DNA sequences of each ASV after denoising.

qiime feature-table tabulate-seqs \
  --i-data deblur/rep-seqs-deblur.qza \
  --o-visualization visualizations/rep-seqs.qzv

Taxonomic classification

This step requires a powerful machine. Use Clover or Blarney to run the following steps. Choose an appropiate reference database based on your samples.

qiime feature-classifier classify-sklearn \
  --i-classifier /mnt/HDD/shyan/2024.09.backbone.v4.nb.sklearn-1.4.2.qza \
  --i-reads deblur/rep-seqs-deblur.qza \
  --p-n-jobs 30 \
  --o-classification taxonomy_greengenes_deblur.qza

Generate visualization of the resulting mapping from sequence to taxonomy

qiime metadata tabulate \
   --m-input-file taxonomy_greengenes_deblur.qza \
   --o-visualization visualizations/taxonomy_greengenes_deblur.qzv

View the taxonomic composition of the samples by generating an interactive bar plots

qiime taxa barplot \

  --m-metadata-file metadata-fruit-2.tsv \
  --o-visualization visualizations/taxa-bar-plot_unfiltered_deblur.qzv

Filtering out Unassigned, d__Bacteria

qiime taxa filter-table \
  --i-table deblur/table-deblur.qza \
  --i-taxonomy taxonomy_greengenes_deblur.qza \
  --p-mode exact \
  --p-exclude "Unassigned,d__Bacteria" \
  --o-filtered-table table-deblur-filtered.qza

Checking taxonomic composition after filtering

qiime taxa barplot \
  --i-table deblur/table-deblur-filtered.qza \

  --m-metadata-file metadata-fruit-2.tsv \
  --o-visualization visualizations/taxa-bar-plot_filtered_deblur.qzv

Checking stats after filtering (i.e. number of sequences per sample)

qiime feature-table summarize \
  --i-table deblur/table-deblur-filtered.qza \
  --o-visualization visualizations/table-summary_filtered.qzv \
  --m-sample-metadata-file metadata-fruit-2.tsv

Filtering out positive and negative controls (i.e. H2O, Zymo, etc.) To do this, create a .txt file that lists the Sample IDs to remove, with one Sample ID per line. This will be used in the –m-metadata-file line.

qiime feature-table filter-samples \
  --i-table deblur/table-deblur-filtered.qza \
  --m-metadata-file control_samples_to_remove.txt \
  --p-exclude-ids \
  --o-filtered-table deblur/table-filtered_excl-unknbact_ctrls.qza

Checking the stats after positive and negative removal.

qiime feature-table summarize \

  --o-visualization visualizations/sampling-depth-table.qzv \

If there’s not a clear jump in feature counts between samples, use a rarefaction curve to determine the best place – choose a max-depth close to the median feature count.
